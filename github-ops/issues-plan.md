# Issues Plan

## Daily issue design standard
Each issue includes:
1. Title
2. Objective
3. Tasks
4. Mini deliverable
5. Reflection question
6. Optional stretch goal
7. Checklist

## Work rhythm
- Open 1 issue/day
- Close only after artifact is committed
- Add short end-of-day comment with key insight and next risk question
